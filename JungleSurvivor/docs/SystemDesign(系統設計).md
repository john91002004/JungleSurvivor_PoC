# JungleSurvivor 系統設計

## 設計哲學

> **在求生場景中，誤判比不判更危險。寧可錯判可食為不確定，絕不錯判有毒為可食。**

系統的每一層設計都遵循「安全第一、保守策略」原則。

## 一、核心架構：分層辨識系統

```
使用者輸入（照片 / 多張照片 / 語音 / 文字描述）
    │
    ▼
┌─────────────────────────────────────────────┐
│  第零層：環境上下文引擎（Context Engine）       │
│  根據 GPS、海拔、氣候帶、季節                   │
│  自動載入該地區專屬的知識庫子集                   │
│  → 大幅縮小搜尋範圍，提高辨識效率和準確度         │
└───────────────────┬─────────────────────────┘
                    ▼
┌─────────────────────────────────────────────┐
│  第一層：危險物種快篩（Danger Screening）       │
│  比對該地區 30-50 種已知有毒/危險動植物          │
│  → 命中任何一種（相似度 ≥ 60%）→ 🔴 立即警告    │
│  → 全部未命中 → 進入第二層                     │
└───────────────────┬─────────────────────────┘
                    ▼
┌─────────────────────────────────────────────┐
│  第二層：易混淆物種專項鑑別（Confusion Pairs）   │
│  針對外觀相似但一個有毒一個可食的物種對           │
│  逐項比對鑑別特徵                               │
│  → 無法確定 → ⚠️ 視為有毒                     │
│  → 確定可食/藥用 → 進入第三層                   │
└───────────────────┬─────────────────────────┘
                    ▼
┌─────────────────────────────────────────────┐
│  第三層：可利用資源辨識（Useful Resources）      │
│  比對該地區 50-100 種常見可食用/藥用植物          │
│  → 信心度 ≥ 70% → 🟢 顯示用途和使用方式        │
│  → 信心度 < 70% → ⚠️ 不確定，建議不碰觸       │
└───────────────────┬─────────────────────────┘
                    ▼
┌─────────────────────────────────────────────┐
│  第四層：通用特徵描述與記錄（Description）       │
│  輸出形態學描述，GPS 標記，存入本地紀錄           │
│  → 有網路後可查詢專業 API 進一步確認             │
└─────────────────────────────────────────────┘
```

## 二、第零層：環境上下文引擎（最關鍵的效率提升）

### 為什麼需要？

全球植物有 40 萬種以上，不分區域地全部比對既慢又不準。
沙漠裡不會出現北極苔蘚，熱帶雨林裡不會出現仙人掌。
**根據所在位置自動縮小搜尋範圍，是整個系統效率和準確度的基石。**

### 輸入來源


| 資訊          | 來源                 | 作用                   |
| ----------- | ------------------ | -------------------- |
| **GPS 座標**  | 裝置 GPS（離線也可用）      | 判斷所在國家/地區            |
| **海拔**      | GPS + 預載地形資料       | 判斷植被帶（平地/山地/高山）      |
| **氣候帶**     | 預載資料（根據緯度/經度/海拔推算） | 判斷熱帶/亞熱帶/溫帶/寒帶       |
| **季節/月份**   | 裝置時鐘               | 判斷植物的生長階段（開花？結果？枯萎？） |
| **使用者手動選擇** | App 設定             | 備用方案，例如 GPS 失效時      |

## 三、第一層：危險物種快篩

> **測試結論（T7）**：用姑婆芋實測，模型給出 65% 相似度並觸發警告。
> 比對模式有效，但特徵描述太簡略導致信心度偏低。
> 改用詳細特徵描述後預期可提升到 80%+。

```

### 物種清單格式（v2 — 詳細特徵版）

> **測試結論**：簡略描述（一行）→ 信心度 65%。
> 詳細描述（含葉形/葉脈/葉面/葉柄/環境）→ 預期信心度 80%+。

每種有毒物種必須包含以下完整資訊，才能讓模型有效比對：

```

   葉形：大型心形或箭形，長 50-90cm，寬 40-60cm
   葉面：光滑發亮，深綠色，水珠不成珠狀（會攤平）
   葉脈：明顯的羽狀脈，主脈粗壯突出
   葉尖：朝上或水平指向
   葉柄：粗壯，長 50-100cm，綠色常帶紫暈，接在葉片邊緣
   莖部：粗短直立莖，多汁
   生長環境：林下、溪邊、潮濕陰暗處，海拔 0-1500m
   易混淆：芋頭 (Colocasia esculenta)

2. 海檬果 (Cerbera manghas)
   毒性：果實劇毒，含海檬果毒素，誤食可致命（心臟毒性）

   葉面：革質，深綠色，有光澤

   花：白色五瓣花，中心粉紅，有香味

3. 曼陀羅 (Datura stramonium)
   毒性：全株有毒，含莨菪鹼，誤食致幻覺、瞳孔放大、可致命
   葉形：卵形，葉緣有不規則粗鋸齒或裂片，長 8-20cm
   葉面：暗綠色，有臭味
   花：白色或淡紫色喇叭形大花，長 6-10cm
   果實：球形蒴果，外被硬刺
   生長環境：荒地、路邊、廢棄地

4. 咬人貓 (Urtica thunbergiana)
   危險：觸碰致劇烈灼痛，持續數小時

   葉面：兩面密生透明焮刺毛（關鍵特徵！）

## 四、第二層：易混淆物種專項鑑別

> **測試結論（T9）**：芋頭和姑婆芋的照片都被正確判定為「無法確定，視為有毒」。
> 安全機制完美運作。但同時揭示了 **靜態照片的物理限制** ——
> 水珠測試等關鍵鑑別手段無法從照片中獲得，必須引導使用者做互動式測試。

### 為什麼需要獨立一層？

有些植物外觀極度相似，但一個可食一個致命。通用辨識無法區分，
需要 **針對特定鑑別特徵逐項比對** 才能判斷。

### 混淆物種對清單（台灣範例）


| 可食/安全                        | 有毒/危險                          | 致命性 | 關鍵鑑別點           |
| ---------------------------- | ------------------------------ | --- | --------------- |
| 芋頭 (Colocasia esculenta)     | 姑婆芋 (Alocasia odora)           | 高   | 水珠測試、葉尖方向、葉柄接合處 |
| 高大環柄菇 (Macrolepiota procera) | 綠褶菇 (Chlorophyllum molybdites) | 中   | 菌褶成熟後是否變綠       |
| 山蘇 (Asplenium nidus)         | 有毒蕨類                           | 低   | 葉形、孢子囊位置        |
| 野薑花 (Hedychium coronarium)   | 有毒薑科植物                         | 中   | 花色、根莖氣味         |
| 雞肉絲菇 (Termitomyces)          | 白色毒菇類                          | 高   | 菌環、生長基質（白蟻巢）    |


### 專項鑑別 Prompt 設計

```
這張照片中的植物可能是以下兩種之一，它們外觀非常相似，但一個可食/安全，一個有毒/危險。

**可食/安全：{安全物種名} ({學名})**
辨識特徵：
{逐條列出安全物種的關鍵特徵}

**有毒/危險：{危險物種名} ({學名})**
辨識特徵：
{逐條列出危險物種的關鍵特徵}

**鑑別要點對照表：**
{插入鑑別特徵對照表}

請根據照片中可辨識的特徵逐項比對，判斷：
1. 更可能是哪一種？列出支持的證據
2. 信心度（0-100%）
3. 哪些特徵無法從照片中判斷？

如果信心度 < 80%，或有任何關鍵特徵無法確認，
請明確標注：「⚠️ 無法確定，視為有毒，禁止食用/接觸」

用繁體中文回覆。
```

### 互動式鑑別流程（T9 測試後新增）

> **設計背景**：T9 測試證明，芋頭和姑婆芋僅靠靜態照片無法區分，
> 因為最關鍵的鑑別手段（水珠測試）需要使用者實際操作。
> 系統必須從「被動辨識工具」進化為「主動求生教練」。

當第二層偵測到混淆物種對時，觸發以下互動流程：

```
Step A：初步照片比對
  → 辨識出混淆物種對（如芋頭 vs 姑婆芋）
  → 判定「僅靠照片無法確定」

Step B：引導使用者做互動式測試
  系統輸出：
  「⚠️ 此植物可能是芋頭（可食）或姑婆芋（有毒），
    僅靠照片無法區分。請執行以下測試：

    🔬 水珠測試（最關鍵！）：
      在葉面滴一滴水，觀察水珠的形狀：
      → 水珠呈圓珠狀，可以滾動 → 較可能是芋頭
      → 水珠攤平，不成珠狀 → 較可能是姑婆芋

    📷 補拍以下照片：
      1. 葉面特寫（觀察是否光滑如蠟？還是有微絨毛？）
      2. 葉柄與葉片接合處特寫（葉柄接在邊緣？還是內側？）
      3. 水珠測試的結果照片

    完成後上傳，我將結合新資訊重新判斷。
    ⚠️ 在完成測試前，請勿食用此植物。」

Step C：接收使用者回饋，重新判斷
  → 使用者回報水珠測試結果（文字或照片）
  → 使用者上傳補拍照片
  → 模型綜合所有新資訊，給出更高信心度的判斷
  → 如果信心度仍 < 80%，維持「禁止食用」
```

每對混淆物種都需要預設一組專屬的「互動式測試指引」：


| 混淆物種對        | 互動測試   | 操作方式          |
| ------------ | ------ | ------------- |
| 芋頭 vs 姑婆芋    | 水珠測試   | 葉面滴水觀察是否成珠    |
| 高大環柄菇 vs 綠褶菇 | 菌褶顏色觀察 | 切開觀察菌褶是否帶綠色   |
| 雞肉絲菇 vs 白色毒菇 | 生長基質觀察 | 是否長在白蟻巢上      |
| 野薑花 vs 有毒薑科  | 根莖氣味測試 | 小心刮開根莖，聞是否有薑味 |

## 五、多張照片策略

### 為什麼需要多張照片？

單張照片經常不夠：


| 問題      | 多張照片如何解決      |
| ------- | ------------- |
| 看不到葉背   | 要求拍攝葉面+葉背     |
| 看不到花/果實 | 要求拍攝不同部位      |
| 看不到整體形態 | 要求拍攝全株+特寫     |
| 光線/角度不佳 | 多角度拍攝降低誤判     |
| 比例不明    | 放參照物（手、硬幣）在旁邊 |


### 引導式拍攝流程

```
系統：「請依照以下順序拍攝照片，以獲得最準確的辨識結果：」

📷 第 1 張：全株照片（從 1-2 公尺距離拍攝整株植物）
📷 第 2 張：葉面特寫（清楚顯示葉脈紋路）
📷 第 3 張：葉背特寫（部分物種的葉背有辨識特徵）
📷 第 4 張：莖部/葉柄特寫
📷 第 5 張：花或果實特寫（如果有的話）
📷 可選：在葉面滴一滴水（芋頭 vs 姑婆芋的水珠測試）
```

### 多照片整合 Prompt

```
以下是同一株植物的多張照片，分別拍攝了不同部位。
請綜合所有照片進行辨識，注意每張照片提供的不同資訊。

照片 1（全株）：整體形態、生長環境、大小
照片 2（葉面）：葉形、葉脈、葉緣
照片 3（葉背）：葉背顏色、毛茸、特殊結構
照片 4（莖部）：莖的顏色、質地、是否有汁液
照片 5（花/果）：花的顏色、形態、氣味描述（如使用者提供）

請綜合以上所有照片的資訊進行判斷。
單張照片看不出的特徵，可以從其他照片補充。
只有當所有關鍵特徵都一致指向同一物種時，才給出高信心度。
```

### 通用優化技巧


| 策略                   | 說明                         | 效果                  | 驗證狀態              |
| -------------------- | -------------------------- | ------------------- | ----------------- |
| **Chain of Thought** | 要求模型「先描述觀察到的特徵，再做判斷」       | 減少跳躍式錯誤結論           | ✅ T8 驗證有效         |
| **環境上下文注入**          | 告訴模型「你在台灣海拔 800m 的亞熱帶闊葉林」  | 大幅縮小候選範圍            | ✅ T8 驗證有效         |
| **安全閾值設定**           | 明確規定不同層級的信心度閾值             | 避免假陽性               | ✅ T5, T6, T9 驗證有效 |
| **負面指令**             | 「絕對不要猜測」「寧可說不知道」           | 抑制模型的「討好」傾向         | ✅ T5, T6, T9 驗證有效 |
| **結構化輸出**            | 要求 JSON 格式輸出（需搭配 CoT，方案 C） | 方便程式後續處理            | ✅ T10-v2 驗證有效     |
| **英文 Prompt + 中文回覆** | Prompt 主體用英文，回覆要求繁體中文      | 平衡準確度和可讀性           | ✅ T8 間接驗證         |
| **詳細特徵描述**           | 知識庫中每種物種包含完整形態特徵           | 信心度 65% → 85% → 95% | ✅ T8, T11 驗證有效    |
| **多照片整合**            | 多張不同角度照片綜合判斷               | 提升 5-10% 信心度        | ✅ T11 驗證有效        |
| **蛇類/動物辨識**          | 與植物共用同一套 Prompt 架構         | 架構泛用性確認             | ✅ T14 驗證有效        |
| **安全第一流程**           | 先查危險 → 排除 → 再查有用           | 確保不漏報               | ✅ T15 驗證有效        |


### ⚠️ 禁止策略（測試中發現的陷阱）


| 策略                    | 為什麼不行                          | 測試依據    |
| --------------------- | ------------------------------ | ------- |
| **封閉式通用辨識**           | 給 N 個選項讓模型選 → 模型硬套答案，即使照片不在選項中 | T3, T4  |
| **過少的特徵描述**           | 只寫一行描述 → 模型比對依據不足，信心度偏低        | T7      |
| **純 JSON 輸出（無 CoT）**  | 跳過思考過程 → 辨識準確度大幅退化（辨錯物種卻給 80%） | T10-v1  |
| **max_new_tokens 太小** | 512 對中文不夠 → 回覆被截斷              | T3 回覆截斷 |

## 七、系統完整流程圖

```
使用者打開 App
    │
    ├─→ 系統讀取 GPS + 海拔 + 時間
    │   → 自動載入對應地區的知識庫
    │   → 顯示「目前環境：台灣，亞熱帶闊葉林，海拔 800m，春季」
    │
    ▼
使用者選擇模式
    │
    ├─→ 📷 「這是什麼？」（通用辨識模式）
    │   → 引導拍攝多張照片
    │   → 進入四層辨識流程
    │
    ├─→ 🔴 「這個有毒嗎？」（安全快篩模式）
    │   → 直接進入第一層危險物種比對
    │   → 快速回覆「危險/安全/不確定」
    │
    ├─→ 🍽️ 「這個能吃嗎？」（食物辨識模式）
    │   → 先過第一層（排除毒物）
    │   → 再進入第二層（混淆物種鑑別）
    │   → 最後第三層（可食性判斷）
    │
    ├─→ 💊 「我受傷了，周圍有什麼能用？」（求生掃描模式）
    │   → 要求拍攝 360 度環境照
    │   → 一次辨識多種植物
    │   → 依「傷口/食物/水源/庇護」分類輸出
    │
    └─→ 🐍 「這是什麼動物？」（動物辨識模式）
        → 辨識物種 + 危險性判斷
        → 如果有毒：急救指引
        → 如果可食：處理方式
```
