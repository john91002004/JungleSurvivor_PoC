# JungleSurvivor：叢林求生離線 AI 助手

## 產品定位

一套運行在手持裝置上的 **離線優先 AI 求生助手**，結合 Gemma 4 的多模態辨識與原生函式呼叫能力，讓孤身處於叢林中的人員（軍人、登山客、野外研究員、災害受困者）能透過「拍照/說話」獲得即時的求生指引。

> **「讓每一個孤身在叢林中的人，口袋裡都有一個經驗最豐富的求生專家、一個野外醫生、一個植物學家、和一個導航員。」**

---

## 為什麼這個場景威力特別強？


| 現實情境         | 為什麼 AI 有價值               |
| ------------ | ------------------------ |
| 與部隊失聯，身處陌生叢林 | 沒有老兵帶路、沒有參考手冊、沒有網路可查     |
| 受傷且沒有醫療兵     | 必須用周遭環境中的東西自救            |
| 口糧耗盡         | 必須辨識什麼能吃、什麼有毒            |
| 水源不確定        | 喝錯水可能比不喝更危險              |
| 需要隱蔽行動       | 不能生大火、不能發出噪音，一切求生行為有限制條件 |

## 完整應用場景

### 一、醫療求生類


| 場景          | AI 做什麼                     |
| ----------- | -------------------------- |
| **傷口處理**    | 辨識周遭可用的止血/消炎/抗菌植物，教你怎麼處理   |
| **骨折/扭傷固定** | 根據環境建議可用的材料製作夾板，指導固定步驟     |
| **蛇咬/蟲螫處理** | 辨識咬傷類型→判斷是否有毒→給出正確急救流程     |
| **發燒/感染**   | 辨識具有解熱、消炎作用的植物→給出用量和使用方式   |
| **腹瀉/水土不服** | 辨識止瀉植物或建議活性炭替代方案（燒木炭）      |
| **傷口縫合替代**  | 辨識可用作天然縫線的植物纖維或昆蟲（如兵蟻夾合傷口） |


### 二、食物與水源類


| 場景          | AI 做什麼                    |
| ----------- | ------------------------- |
| **可食用植物辨識** | 拍照→辨識→告知可食部位、是否需要處理       |
| **有毒植物警告**  | 拍照→紅色警告→說明中毒症狀和誤食處理       |
| **可食用昆蟲辨識** | 辨識蟋蟀、螞蟻、幼蟲等→告知營養價值和處理方式   |
| **菇類安全判斷**  | 極保守策略：信心度不夠高時，一律建議「不要碰」   |
| **水源評估**    | 拍攝水源環境→判斷風險等級→建議淨化方式      |
| **陷阱/捕獵建議** | 根據辨識到的動物足跡，建議適合的陷阱類型和放置位置 |


### 三、環境判讀類


| 場景        | AI 做什麼               |
| --------- | -------------------- |
| **方位判斷**  | 拍攝樹木苔蘚生長面/太陽位置→推算方位  |
| **天氣預判**  | 拍攝雲層/感測氣壓變化→預測短期天氣   |
| **地形分析**  | 拍攝周遭地形→建議移動路線        |
| **水源尋找**  | 辨識指示植物→引導找水          |
| **庇護所選址** | 分析地形、風向、排水→建議搭建位置和方式 |


### 四、威脅偵測類


| 場景           | AI 做什麼                  |
| ------------ | ----------------------- |
| **危險動物聲音辨識** | 收音→辨識蛇類移動聲、大型動物腳步聲、蜂群嗡鳴 |
| **有毒生物辨識**   | 拍照→辨識毒蛇、毒蛙、毒蜘蛛→安全距離建議   |
| **危險植物辨識**   | 辨識觸碰即引起皮膚灼傷的植物          |
| **環境聲音異常偵測** | 持續監聽→區分自然背景音與人造聲音       |

## 深入 User Story

### 場景設定

> **阿凱**，一名步兵，在叢林巡邏中遭遇伏擊後與小隊失散。他左前臂被碎片劃傷（中度撕裂傷，約 8 公分），急救包只剩紗布和一條繃帶。身上還有半壺水、一把刀、一個裝有 JungleSurvivor 的加固型手機（低功耗模式可撐 72 小時）。

### User Story 1：我受傷了，周圍有什麼能幫我？

阿凱包紮了傷口但擔心感染。他打開 App，拍了周圍 5 張照片（360 度環顧）。系統自動辨識出視野中的植物，依「求生用途」自動分類，找到車前草（抗菌消炎）、薑黃屬植物（消炎），並警告附近有咬人貓。同時產出複合處方建議。

### User Story 2：我需要吃東西，什麼能吃？

18 小時沒進食。系統掃描周圍植物，找到可安全食用的山蘇嫩葉、野木耳（需煮熟）、螞蟻/白蟻，並警告疑似姑婆芋（信心度不足，建議不碰），標記海芋屬（有毒）。同時計算可獲取熱量。

### User Story 3：我需要水，但不確定水源安全

發現小溪，系統評估風險為中等，依現有資源排序建議淨水方案（煮沸 > 簡易過濾+紫外線 > 植物過濾法），並指引附近竹子可作為容器。

### User Story 4：解熱消炎 — 複合草藥處方

第二天發低燒，系統判斷傷口感染初期，掃描周圍找到野生薄荷和大花咸豐草，產出「外用抗菌 + 內服退燒」的複合處方，並明確警告這是緊急措施不是醫療替代。

### User Story 5：我要移動，哪條路最安全？

系統根據 GPS + 預載地形圖建議路線，預測沿途資源分布，啟動環境聲音持續監聽模式，並給出電量管理建議。

## MVP 起步策略

1. **最小 MVP**：拍照→辨識植物→顯示「可食用/有毒/藥用」
2. 加上安全警告：毒蛇、毒菇、危險植物即時警告
3. 加上複合處方：根據症狀 + 周遭植物建議處置方案
4. 加上自動記錄：GPS、時間、辨識結果自動歸檔
5. 加上環境音監聽：背景持續分析聲音
6. 加上路線規劃：結合地形圖的移動建議
